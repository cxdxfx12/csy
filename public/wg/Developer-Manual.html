JSON Response

BaseController 核心方法

abstract class BaseController {
    // 成功响应
    protected function success($msg='ok', $data=[], $code=0): Json

    // 错误响应
    protected function error($msg='error', $data=[], $code=1): Json

    // 分页表格数据 (count+列表)
    protected function table($count, $list): Json { return {count,list}; }

    // 通用分页查询
    protected function page($model, $fields='*'): array { page,pageSize }

    // 构建查询条件
    protected function buildWhere($model, $searchFields, &$params): void
    // 搜索修饰符: =精确, %like%模糊, >大于, <小于, [1,2,3] in查询
}

Request 请求修饰符

// 控制器中获取参数:
$this->request->param('keyword');                  // 自动trim
$this->request->param('keyword/s');               // 转为字符串
$this->request->param('page/d');                    // 转为整数 (int)
$this->request->param('amount/f');                  // 转为浮点数 (float)
$this->request->param('ids/a');                     // 转为数组 (array)
$this->request->param('status', '1');             // 带默认值
💡
修饰符: /s=string, /d=int, /f=float, /a=array。在 buildWhere 中自动处理模糊搜索(= 精确, {} like, [] in, > < 范围)。

Axios 请求封装 (request.ts)

// 请求拦截器: 自动注入 Token + 社区ID
service.interceptors.request.use(config => {
  config.headers['token'] = useUserStore().token
  config.headers['community-id'] = useAppStore().communityId
  return config
})

// 响应拦截器: 统一错误处理
service.interceptors.response.use(response => {
  const { code, msg } = response.data
  if (code !== 0) { // 业务错误
    ElMessage.error(msg || '请求失败')
    return Promise.reject(msg)
  }
  return response.data
}, error => {
  if (error.response?.status === 401) {
    // Token 过期 → 清除状态 → 跳转登录
    useUserStore().logout()
    router.push('/login')
  }
  return Promise.reject(error)
})

Nginx 配置要点

server {
    listen 443 ssl;
    server_name www.hbdxm.com;

    root /www/wwwroot/www.hbdxm.com/public;
    index index.html index.php;

    # Vue Router history 模式
    location /admin/ {
        try_files $uri $uri/ /admin/index.html;
    }

    # PHP 反向代理
    location /api/ {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }
}

二次开发快速上手

Quick Start for Developers

1. 克隆并安装

git clone git@github.com:cxdxfx12/csy.git
cd csy
composer install          # PHP依赖
cd admin && npm install    # 前端依赖

2. 添加新页面 (以「公告管理」为例)

// ① 后端：创建控制器 app/admin/controller/Notice.php
class Notice extends BaseAdmin {
    public function list() { /* 分页查询 */ }
    public function add()  { /* 新增 */ }
    public function edit() { /* 编辑 */ }
    public function delete() { /* 删除 */ }
}

// ② 路由：在 route/admin.php 添加
Route::get('admin/notice/list', 'admin/Notice/list');
Route::post('admin/notice/add', 'admin/Notice/add');
Route::post('admin/notice/edit', 'admin/Notice/edit');
Route::post('admin/notice/delete', 'admin/Notice/delete');

// ③ 前端：创建 admin/src/views/notice/NoticeIndex.vue
// 使用 PageTemplate 组件包裹，快速搭建 CRUD 表格页

// ④ 在 router/index.ts 注册路由
{ path: 'notice/index', name: 'NoticeIndex',
  component: () => import('@/views/notice/NoticeIndex.vue'),
  meta: { title: '公告列表' } }

// ⑤ 构建部署
npm run build && node _deploy.js
💡
遵循约定优于配置：Controller 方法名 = list/add/edit/delete/status/export。路由格式 = 模块/controller/方法。

3. 开发规范要点

  • 命名规范：Controller 大驼峰，方法小驼峰；Vue 组件大驼峰 .vue 文件
  • API路径：统一 /api 前缀，模块/控制器/方法 三段式
  • 响应格式：{code:0, msg:"ok", data:{}} 成功码为0
  • 分页查询：统一通过 page/pageSize 参数，返回 {count, list}
  • 小区隔离：非超管用户只能看到自己分配的小区数据
  • Token传递：Header 中携带 token 字段
  • 日志记录：所有 POST/PUT/DELETE 操作自动写入操作日志
  • XSS防护：使用 think-orm 的参数绑定，前端 v-html 需谨慎
