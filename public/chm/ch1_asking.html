CHAPTER 1

提问的艺术：如何让 AI 一次给出正确的代码

80% 的「AI 不好用」问题，都源于不会提问。这一章将彻底改变你的提问方式。

❌ 常见错误提问 vs 正确提问

❌ 糟糕的提问

  • "帮我写一个登录"
  • "这段代码有问题，帮我改"
  • "做一个商城系统"
  • "报错了，怎么办"

✅ 优秀的提问

  • "用 Node.js + MySQL 写一个 JWT 登录接口，包含注册、登录、刷新 Token"
  • "以下代码在并发请求时出现数据竞争，报错信息是..."
  • "请为我设计一个B2C电商系统的数据库ER图，核心模块包括..."
  • "MySQL 8.0 插入时报 Duplicate entry，这是我的表结构和SQL..."

📐 黄金提问公式：CTFS 法则

每一个高质量提问都应该包含以下四个要素：


要素含义示例
C Context（上下文） — 技术栈、环境、约束 "Node.js 18 + TypeScript + PostgreSQL"
T Task（任务） — 具体要做什么 "实现用户角色权限中间件"
F Format（格式） — 期望的输出形式 "请给出完整代码 + 单元测试"
S Scope（范围） — 边界和注意事项 "不考虑并发，Admin角色拥有所有权限"

🎯 实战演示：从烂提问到好提问的进化

Level 0 — 纯新手👶

"帮我写一个后台管理系统"

→ AI 不知道语言、框架、功能、规模，只能给泛泛之谈

Level 1 — 有技术栈但不清晰🤔

"用 Vue 做一个后台管理系统，能管理用户和订单"

→ 给了框架，但缺少细节：用什么 UI 库？路由怎么配？API 格式？

Level 2 — CTFS 完整提问 🔥

请帮我搭建一个 Vue 3 + TypeScript 的后台管理系统基础框架： 【技术栈】 - 前端：Vue 3 + TypeScript + Vite + Pinia + Vue Router - UI库：Element Plus - 后端已有 REST API（Base URL: /api/v1） 【需要实现】 1. 登录页面（用户名+密码+验证码） 2. JWT Token 存储和自动刷新 3. 路由守卫（未登录跳转登录页） 4. 侧边栏菜单（根据角色动态显示） 5. 用户列表页（分页+搜索+增删改查） 6. 订单列表页（分页+筛选+导出Excel） 【要求】 - 项目结构清晰，组件可复用 - 封装统一的 axios 请求实例（拦截器处理Token和错误） - 使用 Pinia 管理全局状态 - 请输出完整的文件结构和每个文件的代码
这就是 CTFS 的威力 当你把上下文、任务、格式、范围都说清楚了，AI 就能一次性输出可直接运行的完整项目代码，而不是让你来回追问 10 次。

🔑 不同场景的提问模板

🆕 新建功能

请用 [语言/框架] 实现 [功能名称]，要求： - 输入参数：[类型和说明] - 输出结果：[类型和说明] - 边界处理：[异常情况说明] - 请包含错误处理和日志记录

🔧 修改现有代码

以下是我当前的代码： [粘贴代码] 我需要做以下修改：[描述修改点] 注意保持其他逻辑不变，尤其是 [关键部分]

📖 技术咨询

我在做 [项目描述]，技术栈是 [技术栈] 现在面临 [具体问题/场景] 我考虑了以下方案：[A、B、C] 请帮我分析各方案的优劣，并给出推荐

🐛 调试求助

环境：[OS + 语言版本 + 框架版本] 复现步骤：[1. 2. 3.] 期望行为：[应该怎样] 实际行为：[实际怎样 + 完整报错信息] 相关代码：[精简后的最小可复现代码] 已尝试的解决方案：[A、B、C — 都无效]

🧪 进阶技巧：多轮对话策略

你
我想做一个小区物业管理系统，能帮我设计数据库吗？
AI
好的！为了更好地设计，请告诉我：
1. 需要管理哪些核心业务（业主、缴费、报修？）
2. 是多小区还是单小区？
3. 是否有微信小程序端？
你
单小区模式，核心业务：业主管理、物业费收缴、报修工单、停车管理、门禁。有微信小程序端。
AI
明确了！我来设计一套完整数据库方案，包含 15 张核心表...
策略要点 第一轮给出大方向让 AI 追问细节，第二轮补充完整，一次性拿到成熟方案。不要试图一句 Prompt 解决所有问题。
