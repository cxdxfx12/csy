**任务2**: 修复 `Class "Firebase\JWT\JWT" not found`

**根因**: 服务器 `vendor/composer/installed.json` 为空，vendor 文件是手动拷贝而非 composer install 安装的。autoload_psr4.php 只注册了 app/middleware/service/think 四个命名空间，所有第三方包（firebase/phpoffice 等）都没注册。

**修复方式**: 扫描 vendor 目录，直接将 `Firebase\JWT\` 路径注入 `autoload_psr4.php`。composer install 无法运行（fileinfo 扩展缺失 + 安全策略阻止 firebase/php-jwt）。

**补充-根因纠正**: 仅修复 `autoload_psr4.php` 不够！Composer v2 使用 `autoload_static.php` 作为主加载源。ClassLoader 优先从 `$prefixLengthsPsr4` 和 `$prefixDirsPsr4`（在 autoload_static.php 中）查找类，而非 autoload_psr4.php。最终通过改写 autoload_static.php 的 prefixLengthsPsr4 和 prefixDirsPsr4 添加 Firebase 条目，并重启 PHP-FPM 清空 OPcache 后生效。CLI 验证 class_exists("Firebase\\JWT\\JWT") = SUCCESS。

**服务器关键状态**:
- PHP-FPM 8.1 路径: /www/server/php/81/sbin/php-fpm, conf: /www/server/php/81/etc/php-fpm.conf
- Socket: /tmp/php-cgi-81.sock
- FPM init: /etc/rc.d/init.d/php-fpm-81（经常需要手动 kill 僵尸进程再启动）
- fileinfo 扩展缺失但项目仍能运行
- 网站 vendor 是手动拷贝，installed.json 为空，composer install 不可用
