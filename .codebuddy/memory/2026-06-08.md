## 硬件设备页面搜索功能修复
- **根因**：`Device.vue` 中 `apiGet` 调用错误地使用了 `{ params }` 嵌套结构。`apiGet(url, { params })` 会导致 axios 发送 `?params[keyword]=xxx`，而后端 `Device.php` 的 `lists()` 读取的是顶层 `keyword` 参数，因此搜索条件永远传不到后端。
- **修复**：将 `apiGet('/admin/equipment/deviceList', { params })` 改为 `apiGet('/admin/equipment/deviceList', params)`，并同步修复了 `deviceActions` 和 `deviceGetStatus` 两处同样的问题。同时修复了 `query.status` clearable 后变为 `null` 导致错误传参的边界情况。
- 已构建并部署到服务器。
- **⚠️ 部署失败排查**：第一次构建部署后，服务器上的JS仍为旧代码 `deviceList",{params:o})`，原因是 zip 文件未重新生成导致部署的是旧版。第二次手动创建新zip并部署后验证成功，新代码 `deviceList",o)` 已生效。**教训：每次 build 后必须重新生成 zip 再部署。**
- 用户确认修复生效，选择小区后搜索可以正确过滤。

## 官网图片丢失问题修复
- **根因**：`welcome-girl.jpg` 和 `monkey-icon.png` 放在 `public/admin/assets/images/` 下，每次 Vite 构建（`emptyOutDir:true`）和部署（`rm -rf assets`）都会被删掉。另外 `monkey-icon.png` 源文件名实际是 `monkey-ico.png`。
- **永久方案**：将官网用到的图片移到 `public/images/`（admin 目录外，不受构建部署影响），`index.html` 中引用路径从 `/admin/assets/images/xxx` 改为 `/images/xxx`。
- 已上传到服务器 `/www/wwwroot/www.hbdxm.com/public/images/`，验证 200 OK。
