3. **数据库**：执行 SQL 修复所有已存在的空 code 行：`UPDATE ds_charge_item SET code = CONCAT('CI_', ...) WHERE code = '' OR code IS NULL`

### 根因
之前的 `_deploy_fix.js` 部署脚本中，远程 SFTP 路径使用了 Windows 风格 `\` 分隔符（`path.join()` 在 Windows 下输出 `\`），导致服务器上文件名包含反斜杠（如 `assets\index-BzGuCcou.js`），而 `index.html` 引用的是正斜杠 `/admin/assets/index-BzGuCcou.js`，浏览器 404。

### 关键教训
**SFTP 远程路径必须使用 `/` 正斜杠**，不能依赖 `path.join()` 在 Windows 上做路径拼接。部署脚本需 `.replace(/\\/g, '/')` 转换路径分隔符。

### 自定义 Query Builder vs 标准 ThinkPHP ORM 差距分析 (18:31-18:36)

**背景**：这个 333 行的自定义 Builder 是我早期搭建项目时为了绕过 think-orm 依赖注入复杂度而手写的，DB 连接配置也硬编码在 getPdo() 中。

### 方案A：补全自定义 Builder (18:37-18:40)

在 `vendor/think/facade/Db.php` 补全 5 项：

1. **`whereOr($field, $op, $val)`** — 与 where() 相同签名，条件存入 `$orWhere[]` 数组
2. **`buildConditions($conditions, $separator)`** — 提取纯条件构建逻辑，支持 AND/OR 分隔符
3. **`buildWhere()` 重构** — 支持 Closure 闭包子查询（修复 IoT/Ai 的关键词搜索），处理 orWhere 用 OR 连接并括起来
4. **`lock($lock=true)`** — 新增 `$lock` 属性，`select()` 末尾追加 `FOR UPDATE`，`find()` 自动继承
5. **`Db::startTrans()/commit()/rollback()`** — Db 类静态事务方法，通过 PDO inTransaction() 安全包装
6. **`__call($name, $arguments)`** — 魔术方法兜底，抛出带方法名的 `BadMethodCallException`
