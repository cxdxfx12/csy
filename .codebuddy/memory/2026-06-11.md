## 修复经理端鉴权绕过高危漏洞 (21:55-21:59)
- **漏洞**：BaseManager::auth() 中用 `$this->error()` 而非 `$this->throwError()`，导致 5 处鉴权失败仅返回 Json 对象而不中止执行

- **根因**：`BaseController::error()` 只 `return new Json(...)`，不像 `throwError()` 抛出 `HttpResponseException` 中止执行。BaseAdmin 和 BaseStaff 均使用 `throwError`，仅 BaseManager 是遗漏
- **修复**：BaseManager::auth() 中 5 处 `$this->error(...)` → `$this->throwError(...)`
- **验证**：修复后 12 个 API 全部返回「请先登录」→ 0 泄露；正常 token 访问不受影响；wechatLogin 假 code 被微信 API 拒绝；wechatRegister 伪造 openid 虽可写入但 status=0 无法获取 token
