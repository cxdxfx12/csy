项目管理：从需求到上线的 AI 全流程

AI 不仅是编码助手，更是你的项目管理搭档

🗺️ AI 辅助项目全生命周期

阶段AI 能做什么你做什么
📋 需求分析 将PRD转为功能清单、画流程图、识别技术风险 确认业务逻辑、排优先级
🏗️ 技术设计 数据库设计、API设计、技术选型建议 审核方案、拍板决策
💻 开发阶段 生成脚手架、CRUD代码、单元测试、修复Bug Review代码、集成调试
🧪 测试阶段 生成测试用例、E2E测试脚本、Bug分析 执行测试、验收确认

📝 文档维护 API文档、README、变更日志、运维手册 审核补充、版本管理

📅 项目启动：让 AI 帮你做计划

Prompt 模板：

我要启动一个 [项目名称] 项目： 【项目概况】 - 目标用户：[描述] - 核心功能：[列举] - 技术栈：[前端 / 后端 / 数据库 / 部署] - 团队规模：[X人] - 时间要求：[X周/月] 【请帮我输出】 1. 开发阶段划分（Phase 1 / 2 / 3，每个Phase的具体任务） 2. 每个Phase的预估工时 3. 技术风险清单和应对方案 4. 推荐的协作流程（分支策略、Code Review规范） 5. 第一周的工作计划（具体到每天）

📊 进度管理：用 AI 写周报和追踪进度

把每天的开发日志喂给 AI，让它自动生成周报。

根据以下本周开发日志，生成一份周报： 【本周日志】 - 周一：完成登录注册模块，修复了Token刷新bug - 周二：开发业主管理CRUD，遇到手机号重复校验问题 - 周三：解决手机号重复bug + 完成楼栋管理模块 - 周四：前后端联调，修复了5个API参数不一致问题 - 周五：部署测试环境，发现跨域问题并解决 【周报格式】 - 本周完成、进行中、下周计划、风险与阻塞 - 每个任务标注完成百分比

🔧 DevOps：AI 生成部署配置

Dockerfile 生成：

请为我的 PHP 8.1 + Nginx 项目生成 Dockerfile： - 使用 php:8.1-fpm-alpine 基础镜像 - 安装 pdo_mysql、redis、gd、zip 扩展 - 用 composer install --no-dev 安装依赖 - 暴露 9000 端口 - 多阶段构建优化镜像大小

Nginx 配置生成：

请为此项目生成 Nginx 配置： - 域名：admin.example.com - 前端静态目录：/var/www/public/ - PHP 请求转发到 php-fpm:9000 - SPA 模式（所有非文件请求回退到 index.html） - 开启 gzip、设置缓存策略 - 安全头（X-Frame-Options、CSP等）

CI/CD 配置：

请为 GitHub Actions 生成 CI/CD 配置： - 代码 push 到 master 时触发 - 步骤：checkout → composer install → phpunit → 构建前端 → SCP部署到服务器 - 部署前发送钉钉通知，部署后发送结果通知

🚀 上线 CheckList（让AI帮你检查）

我即将上线项目，请帮我生成一份上线检查清单，覆盖： 【安全】 - 是否关闭了 debug 模式 - 数据库密码是否使用环境变量 - 敏感接口是否有鉴权 - HTTPS是否配置 【性能】 - 是否开启了 OPcache - 静态资源是否配置CDN - 数据库慢查询是否优化 【可靠性】 - 是否有健康检查端点 - 日志轮转是否配置 - 备份策略是否到位 - 回滚方案是否准备
