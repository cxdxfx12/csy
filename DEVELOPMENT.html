3 后端架构

请求如何被处理 —— 从浏览器到数据库的完整链路。

3.1 请求生命周期

浏览器请求 │ ▼ ┌─────────────────────────────────────────────────┐ │ public/index.php │ │ 1. 加载 .env → $_ENV │ │ 2. CORS头设置 (动态Origin白名单) │ │ 3. 创建 app\Request 对象 │ │ 4. URL路由解析 │ │ ├── 剥离 /index.php/ 或 /api.php/ 前缀 │ │ ├── 处理双重 /api/api/ 前缀 │ │ ├── 路由缓存读取/刷新 │ │ └── Route::get() → 模块/控制器/方法 │ │ 5. 实例化控制器 │ │ ├── 执行中间件链 (middleware.php) │ │ │ ├── CorsMiddleware (跨域) │ │ │ ├── AuthMiddleware (JWT认证+频率限流) │ │ │ └── LogMiddleware (操作日志) │ │ ├── 控制器 initialize() → auth() │ │ └── 控制器 action() │ │ 6. 异常捕获 → 友好错误JSON │ └─────────────────────────────────────────────────┘

9 开发指南

9.1 本地开发启动

# 1. 启动 PHP 后端 cd e:\ds php -S 127.0.0.1:8000 public/index.php # 2. 启动 Admin 前端 (开发模式) cd e:\ds\admin npm install npm run dev # → http://localhost:3000/admin/ # 3. 启动独立前端应用 cd e:\ds\frontend npm install npm run dev # 4. 构建 Admin 前端 cd e:\ds\admin npm run build # → 输出到 ../public/admin/ # 构建脚本会自动复制静态图片到 public/admin/assets/images/ # 5. 数据库 mysql -u root -p < database/dasheng.sql # 然后依次导入其他SQL文件 mysql -u root -p dasheng < database/access_tables.sql mysql -u root -p dasheng < database/gate_tables.sql # 默认管理员: admin / 密码见 .env 配置

9.2 添加新功能的标准流程

1. 建表 └── database/ 目录添加 SQL 或写入 dasheng.sql (表前缀 ds_) 2. 后端控制器 └── app/admin/controller/NewFeature.php extends app\admin\BaseAdmin └── public function lists() → $this->table() └── public function add() → $this->success() └── public function edit() → $this->success() └── public function delete() → $this->success() 3. 注册路由 └── route/admin.php 添加 Route::get/post 规则 4. 权限配置 └── ds_menu 表插入新菜单记录 (填写 permission 字段) └── BaseAdmin.php → buildPermissionMap() 添加映射 └── BaseAdmin.php → getRolePermissions() 添加白名单 5. 前端页面 └── admin/src/views/newfeature/NewFeature.vue ├── 搜索表单 + 数据表格 (Element Plus) └── 增/改弹窗 6. 注册前端路由 └── admin/src/router/index.ts 添加路由配置 7. 构建部署 └── cd admin && npm run build └── 构建完成自动复制 static 资源
